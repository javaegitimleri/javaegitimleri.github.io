SPRING SECURITY FRAMEWORK EĞİTİMİ

Bu eğitimde kurumsal web uygulamalarının çok çeşitli güvenlik ihtiyaçlarına platform bağımsız çözümler sunan Spring Security Framework kapsamlı biçimde anlatılmaktadır. Eğitim sonunda klasik web container security yaklaşımının yetersizliklerini, JAAS ile boğuşmanın anlamsızlığını görecek, bunların yetersizlikleri ve karmaşıklıkları nedeni ile “in-house” üretmek zorunda kaldığınız çözümlerle aslında bir “security framework”e doğru gittiğinizin farkına varacaksınız. Pek çok değişik projede kullanılan, geniş bir kullanıcı kitlesine sahip, olgunlaşmış bir “security framework” ile projenizin başından itibaren güvenlik ihtiyaçlarınıza kolayca çözümler üretmenin rahatlığını ve güvenini keşfedeceksiniz.

Eğitim İçeriği

  • Spring Security Framework'e Giriş
  • Kimliklendirme
  • Cross Site Request Forgery (CSRF) Saldırılarını Önleme
  • HTTP Basic Kimliklendirme
  • Yetkilendirme
  • Spring Security Özelleştirmeleri
Opsiyonel Konular
  • HTTP Digest Kimliklendirme
  • Spring Security ve Event Yönetimi
  • Spring Security ve Request URL
  • Crypto Modülü
  • Spring Security ve HTTP Response Headers
  • Spring Security ve Vaadin Entegrasyonu
İleri Düzey Konular
  • LDAP ile Kimliklendirme
  • X509 Sertifika Tabanlı Kimliklendirme
  • Kayıt Düzeyinde Yetkilendirme (ACL)
  • RunAs Kabiliyeti ile Geçici Yetkilendirme
  • Security Interceptors ile Yetkilendirme
  • Switch User Kabiliyeti
Eğitim süresince ana konuların yanı sıra katılımcıların ihtiyaçlarına, bilgi ve tecrübe düzeylerine, eğitim süresine bakılarak opsiyonel ve ileri düzey konu başlıklarından uygun görülen konulara da değinilmektedir.

Eğitimin Süresi


Eğitim için önerilen süre 3 gündür. Eğitim boyunca konu anlatımları lab çalışmaları ile desteklenmektedir. Lab çalışmaları katılımcılarla birlikte gerçekleştirilmektedir.

Eğitimin Faydaları


Eğitim sonunda katılımcılar;

  • Web uygulamalarında ki güvenlik ihtiyaçlarını
  • Spring Security’nin temel yapıtaşlarını, mimarisini, çalışma prensibini
  • Kimliklendirme ve yetkilendirmenin ne anlama geldiğini
  • Değişik kimliklendirme yöntemlerini
  • Kullanıcı bilgilerinin veritabanı ve LDAP gibi kaynaklardan nasıl alınabileceğini
  • Web kaynaklarının yetkilendirilmesini
  • Servis metotları düzeyinde yetkilendirme yapmayı
  • Domain nesneleri düzeyinde yetkilendirme ihtiyaçlarının nasıl karşılanabileceğini
  • Kriptolu şifreler oluşturmayı
  • Beni hatırla gibi web uygulamalarında sıkça istenen güvenlik fonksiyonlarının nasıl hayata geçirileceğini
  • Çalışma zamanında kullanıcı kimliğini geçici olarak değiştirmeyi
  • Bir kullanıcının aynı anda farklı yerlerden kaç defa oturum açabileceğinin sınırlandırılmasını


ve daha pek çok önemli konuyu öğrenecekler, Spring Security Framework’ü web uygulamalarında kolayca konfigüre ederek kullanabilecek ve projelerine özgü uyarlamalar yapabilecek düzeye geleceklerdir.
